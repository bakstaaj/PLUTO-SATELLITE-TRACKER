# v2.8.10 GMSK windowed timing analyzer

Adds an offline windowed GMSK timing analyzer for PCM s16le captures collected from the Pluto Satellite Tracker live audio path.

## v2.8.10c fast retry

The original windowed analyzer could run too long on a 30+ second capture because it walked too many symbols and phase candidates in pure Python. The v2.8.10c retry replaces that implementation with a bounded fast analyzer that:

- analyzes short windows with a configurable hop size;
- samples a bounded number of symbols per phase/window;
- ranks best timing windows;
- exports the best PCM window for later slicer/HDLC experiments;
- clearly states that no HDLC, AX.25, or text decode is claimed.

Typical use:

```bash
python3 tools/analyze_gmsk_windows_v2_8_10.py \
  ../gmsk-phase2-capture-harness-v2.8.8/captures/gmsk_phase2/gmsk_capture_20260620T201209Z.pcm
```

Outputs are written under `analysis/gmsk_phase2/`.
