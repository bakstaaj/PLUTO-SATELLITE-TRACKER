# v2.8.26 Selected pass row button unlock

This web-only patch keeps pass-row Listen/Receive buttons enabled for backend testing after a row is selected.

## Changes

- Adds a selected-pass row-button unlock layer after `renderPassDetail()`.
- Removes late `disabled` / `aria-disabled` state changes from pass-row action buttons when the row has a visible downlink.
- Adds a MutationObserver to repair the selected row if legacy UI code disables it after selection.
- Adds CSS override for selected row action pointer events and opacity.
- Adds console helpers:
  - `window.plutoUnlockPassRowButtonsV2826()`
  - `window.plutoSchedulePassRowUnlockV2826()`

No backend C or radio API code is changed.
