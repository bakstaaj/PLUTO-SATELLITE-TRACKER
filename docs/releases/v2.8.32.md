# v2.8.32 - Audio live 409 retry for backend-test pass rows

## Summary

Adds a browser-side retry wrapper for `/api/radio/audio/live.wav` HTTP 409 responses seen when testing pass-row Listen/Receive actions outside a live pass.

## Behavior

- Intercepts only `/api/radio/audio/live.wav` responses.
- On HTTP 409, stops stale backend live audio, restarts live audio for the requested downlink, then retries the same stream once.
- Shows recovery/failure status in the backend-test modal or analog audio status area.
- Exposes `window.plutoAudioLive409RetryV2832.reset(downlinkHz)` for field diagnostics.

## Scope

- Web UI only.
- No backend C changes.
