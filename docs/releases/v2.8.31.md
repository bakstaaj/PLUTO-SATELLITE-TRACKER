# v2.8.31 - Force-open Backend Test modal and capture 409 errors

This UI-only patch repairs the pass-row Backend Test flow after v2.8.30.

## Changes

- Force-creates and force-opens the Backend Test modal before any backend audio stream starts.
- Adds authoritative modal overlay styling so the dialog is visible even if earlier modal CSS is missing.
- Intercepts pass-row action clicks at capture phase and routes them into the Backend Test modal.
- Resets stale audio sessions using the existing stop path before starting a listen test.
- Captures live audio start/stream failures, including HTTP 409, inside the modal so the error can be copied.
- Keeps Receive/data rows in the Backend Test modal without reopening the retired legacy Receive placeholder.

## Scope

- Web UI only.
- No backend C changes.
- No decoder endpoint changes.
