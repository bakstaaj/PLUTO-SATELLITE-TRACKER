# v2.8.36 Backend audio bounded stop

UI-only cleanup after the v2.8.34 backend audit.

The audit proved the backend live-audio path works reliably when called in a deterministic order:

1. `/api/radio/audio/live/stop`
2. `/api/radio/audio/live/start?...`
3. `/api/radio/audio/live.wav?...`

The remaining browser issue was that the cleanup step could block indefinitely before the start call. This release makes `/api/radio/audio/live/stop` a bounded cleanup operation in the browser. It tries to clear stale backend audio, but if the stop request times out or fails, it reports the condition and continues to the proven start -> stream sequence.

No backend C code is changed.
