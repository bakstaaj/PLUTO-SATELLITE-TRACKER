# v2.8.38 - Backend audio start verification

This release cleans up the pass-row backend Listen test startup path after the v2.8.34b audit proved that the backend stream works when called in the correct sequence.

## Changes

- Replaces the active `startAnalogAudio()` path with an explicit verified sequence:
  1. Clear any browser-side audio session locally.
  2. Send a bounded `/api/radio/audio/live/stop` cleanup request.
  3. Plan the radio target best-effort.
  4. Send `/api/radio/audio/live/start` with `downlink_hz` and `mode`.
  5. Open `/api/radio/audio/live.wav` only after `live/start` returns HTTP OK and does not report `ok:false`.
- Surfaces the exact `live/start` or `live.wav` response body in the Backend Test modal/status area.
- Does not restore the earlier 409 reset/retry loop.
- Does not touch backend C.
