# v2.8.15 - Audio stream 409 recovery

This web-only update improves the browser live-audio path when the backend returns
HTTP 409 from `/api/radio/audio/live.wav`.

## Changes

- Detects HTTP 409 from the live WAV stream.
- Stops any stale backend audio session.
- Restarts backend audio DSP once and retries the stream.
- Shows a clear operator message instead of a raw 409 URL error.
- Handles 409 during the initial live-audio start request as well as during the stream fetch.

## Safety

- Backend C is not changed.
- Decoder behavior is not changed.
- The update is limited to `web/app.js` plus this release note.
