# v2.8.25 Pass-row direct action handler

This web-only patch makes pass-row Listen/Receive actions usable for backend testing outside a live pass.

## Changes

- Removes the static disabled attribute from the pass-row action button template.
- Overrides `configurePassRowActionButtonV286()` with a direct backend-test click path.
- A pass row action now only requires a downlink; it is no longer gated by AOS/LOS readiness or `isPassTunable()`.
- Selecting a row action selects the pass and then triggers the existing backend-owned audio action.
- If the legacy primary audio button is hidden or missing, the row action calls `startAnalogAudio()` directly.
- Adds `window.plutoRepairPassRowButtonsV2825()` as a browser-console repair helper.

No backend C or radio API code is changed.
