# v2.8.33 - Abort-safe live audio 409 recovery

This UI-only release fixes the pass-row backend-test audio recovery path after an HTTP 409 from `/api/radio/audio/live.wav`.

## Change

- Overrides `startAnalogAudio()` with an abort-safe recovery loop.
- On live WAV HTTP 409, stops stale backend live audio, restarts backend audio DSP, and retries the stream with a new `AbortController`.
- Avoids calling `stopAnalogAudio()` during the 409 retry because that aborts the active fetch controller and can produce `signal is aborted without reason`.
- Surfaces retry and failure messages in the backend-test modal/status area.
- Leaves backend C unchanged.
