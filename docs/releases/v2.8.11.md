# v2.8.11 - GMSK high-rate IQ capture helper

## Purpose

The v2.8.10 windowed analyzer confirmed that the 24 kHz live-audio PCM path sees a real GMSK signal, but every 1-second timing window remained weak.  That means HDLC/AX.25 flag probing should not be attempted on the speaker-audio stream yet.

This release adds a high-rate IQ capture helper so GMSK Phase 2 work can move to the correct signal source: interleaved int16 I/Q samples from the Pluto IIO stream.  The captured IQ file can be used offline for FM/discriminator analysis, symbol-clock recovery, slicer threshold selection, and later HDLC flag/frame experiments.

## Added

- `tools/capture_gmsk_iq_window_v2_8_11.py`
  - Captures high-rate IQ through SSH from Pluto using `iio_readdev`.
  - Stops backend live audio before capture by default to reduce IIO buffer contention.
  - Records `/api/status`, `/api/radio/status`, and `/api/radio/track/state` metadata.
  - Writes `.iq`, `.json`, and `.txt` artifacts under `captures/gmsk_phase2_iq/`.
  - Performs a small preview analysis: I/Q RMS, peaks, DC balance, clipping, and coarse frequency offset.

## Recommended field use

Run this during a GMSK pass after the radio has been tuned/tracked by the UI:

```bash
cd ~/sdrdev/PLUTO-SATELLITE-TRACKER-worktrees/gmsk-high-rate-iq-capture-v2.8.11
python3 tools/capture_gmsk_iq_window_v2_8_11.py --host 192.168.68.104 --seconds 4
```

Four seconds at the default assumed 2.4 MS/s produces roughly 38 MB of IQ data.  Keep captures short until the offline discriminator/clock-recovery workflow is proven.

## Decoder claim

No GMSK, HDLC, AX.25, or text decode is claimed in this release.  This release only creates better raw diagnostic artifacts.

## Backend status

No backend C source is changed.
