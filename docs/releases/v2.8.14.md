# v2.8.14 Pass action literal mode labels

## Purpose

Fix remaining pass-list action-label mismatches by making the action button label follow the visible transmitter mode exactly.

## Corrected examples

- `401.030 MHz GMSK USP` -> `Receive: GMSK USP`
- `145.825 MHz AFSK` -> `Receive: AFSK`
- `145.860 MHz BPSK` -> `Receive: BPSK`
- `10475.000 MHz DVB-S2` -> `Receive: DVB-S2`
- `435.xxx MHz CW` -> `Receive: CW`

## Behavior

The button no longer maps AFSK to APRS, BPSK to Packet, GMSK to Digital, or DVB-S2 to unrelated catalog hints. Voice/audio modes remain `Listen`.

## Scope

- Web UI only.
- No backend C changes.
- No decoder behavior changes.
