# v2.8.29 - Retire legacy Receive Decode placeholder during pass-row backend tests

This UI-only patch prevents duplicate dialogs when using pass-row Listen/Receive buttons for backend testing.

## Changes

- Retires the older `Receive Decode` placeholder modal for pass-row action testing.
- Forces `#receivePlaceholderModalV282` hidden if legacy listeners try to open it.
- Hides and disables the older `#receiveDecodePlaceholderButtonV282` binding.
- Leaves the newer pass-row Backend Test modal as the single operator dialog.
- Adds console helpers:
  - `window.plutoHideLegacyReceivePlaceholderV2829()`
  - `window.plutoRetireLegacyReceivePlaceholderV2829()`

## Scope

- Web UI only.
- No backend C changes.
- No decoder/radio endpoint changes.
