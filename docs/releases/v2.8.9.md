# v2.8.9 GMSK Phase 2 Offline Analyzer

Marker: `GMSK_PHASE2_OFFLINE_ANALYZER_V2_8_9`

This release adds a local offline analyzer for PCM files captured from the GMSK Phase 2 capture harness.

## Purpose

The analyzer helps evaluate captured signed 16-bit little-endian PCM windows before backend demodulator work. It estimates:

- sample count and duration
- RMS, peak, DC offset, clipping count
- samples per symbol for an assumed 9600 symbols/sec path
- zero-crossing estimate
- candidate symbol timing phases
- candidate slicer threshold readiness
- clock/slicer status labels

## Non-goals

This release does not claim packet decode. It does not decode HDLC, AX.25, or text. It is intentionally an offline diagnostic tool.

## Example

```bash
cd ~/sdrdev/PLUTO-SATELLITE-TRACKER-worktrees/gmsk-phase2-offline-analyzer-v2.8.9
python3 tools/analyze_gmsk_pcm_v2_8_9.py \
  ../gmsk-phase2-capture-harness-v2.8.8/captures/gmsk_phase2/gmsk_capture_20260620T201209Z.pcm
```

Outputs are written under:

```text
analysis/gmsk_phase2/
```

## Notes from the first real captures

The first uploaded PCM captures showed healthy RMS/peak levels, no clipping, very low DC offset, and candidate transition activity. They also confirmed that 24 kHz audio provides only 2.5 samples per 9600-symbol period, which is usable for offline diagnostics but tight for robust live demodulation. The next backend step remains a higher-rate discriminator/baseband path before true clock recovery, slicer lock, and HDLC flag/frame counting.
