# Pluto Satellite Tracker v2.7.0

Release date: 2026-06-19

## Summary

v2.7.0 promotes the v2.6.x decoder and receive-mode work into a minor release.  The release keeps FM/voice listening behavior intact while adding a structured decoder foundation for CW, AX.25/APRS, BPSK, FSK and GMSK modes.  It also adds periodic pass refresh from the web UI so the pass queue stays filled while the app is open.

## User-facing changes

- FM, AM, USB, LSB and SSB passes continue to use the normal **Listen** audio path.
- CW, Morse and A1A passes route to **Decode CW**.
- AX.25, APRS, AFSK packet and packet-style passes route to digital **Decode**.
- BPSK, PSK, FSK, GMSK and 9600-style passes route to digital **Decode** instead of CW.
- USB/LSB linear transponder passes keep **Listen** as the primary action and expose a secondary **Decode CW** override for CW inside the passband.
- The decode modal now uses friendly states such as `CW decoder waiting`, `BPSK signal diagnostics`, and `FSK/GMSK signal diagnostics` instead of internal backend/debug state strings.
- The Decoder Tests panel exposes CW, AX.25, AX.25 AFSK, generic digital, BPSK, FSK and GMSK self-tests.
- The web UI periodically queues pass refreshes approximately every 15 minutes while the app is open.

## Decoder foundation

- CW Morse timing self-test is available and returns synthetic `SOS`.
- AX.25/APRS frame parser self-test is available and returns `N0CALL>APRS:>AX25 SELFTEST`.
- AX.25 Bell 202 AFSK synthetic self-test is available.
- BPSK synthetic self-test is available.
- FSK/GMSK synthetic self-tests are available.
- Live BPSK and FSK/GMSK decode output now routes to mode-specific diagnostic states instead of the generic AX.25 path.
