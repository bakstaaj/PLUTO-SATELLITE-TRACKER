# v2.8.37 Backend Audio Direct Start

This UI-only patch bypasses the legacy `stopAnalogAudio()` await path when starting
pass-row backend Listen tests. The prior cleanup could block forever at
"Stopping any previous browser/backend audio session..." and prevent the
proven backend order from reaching `/api/radio/audio/live/start`.

## Behavior

- Clear the browser audio session locally without awaiting the legacy stop path.
- Call `/api/radio/audio/live/stop` as a bounded cleanup only.
- Continue to `/api/radio/audio/live/start` even if cleanup times out.
- Open `/api/radio/audio/live.wav` only after `/live/start` succeeds.
- Do not restore browser-side 409 reset/retry loops.
- Do not modify backend C.

Marker: `BACKEND_AUDIO_DIRECT_START_NO_LEGACY_STOP_V2_8_37`.
