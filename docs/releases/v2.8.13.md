# v2.8.13 Pass action mode label match

## Purpose

Fix pass-list Listen/Receive action labels so they match the visible transmitter mode line shown under each satellite instead of being inferred from satellite-name or broad catalog metadata.

## Behavior

The action label now uses the same mode source that is displayed in the pass row:

- `CW` -> `Receive: CW`
- `AFSK` -> `Receive: AFSK`
- `GMSK` -> `Receive: GMSK`
- `BPSK` -> `Receive: BPSK`
- `DUV` -> `Receive: DUV`
- voice/audio modes such as FM/AM/SSB remain `Listen`

This prevents examples such as a visible `437.200 MHz CW` row from being mislabeled as `Receive: APRS` because the satellite has other APRS/packet metadata elsewhere in the catalog.

## Scope

- Web UI only.
- No backend C changes.
- No decode claims added.
