# v2.8.7 - GMSK Signal Diagnostics Phase 1

Adds a GMSK/9k6-specific diagnostic display layer to the existing Receive/Decode modal.

This release does not claim bit, HDLC, AX.25, or text decode. It keeps the known-good backend receive path intact and makes the existing signal diagnostics easier to interpret for GMSK passes by adding:

- Target symbol-rate assumption for 9600/G3RUH/GMSK passes.
- Samples-per-symbol calculation.
- Level suitability summary from RMS/peak.
- Clock-readiness and slicer-readiness notes.
- Transition-activity interpretation from the existing zero-crossing estimate.
- Clear next-step guidance for backend clock recovery, slicer, and frame flag counting.

Scope: web UI diagnostic display only. Backend C is intentionally unchanged because the deployed 2.6.37a backend binary is currently the known-good receiver runtime and the repo C source is not yet aligned with that binary.
