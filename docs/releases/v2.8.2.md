# v2.8.2 Mode-aware Receive UI Phase 1

This enhancement adds a UI-only Receive foundation for non-voice satellite modes.

Validation scope:

- FM/AM voice passes continue to use the existing Listen audio path.
- CW, packet, APRS, AFSK, FSK, telemetry, and other data-like modes expose a Receive placeholder.
- Backend C is intentionally unchanged in this phase.
- Pluto live status and pass payload validation remain required.
