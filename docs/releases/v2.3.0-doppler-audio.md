# Pluto Satellite Tracker v2.3.0 - Doppler Audio Upgrade

Release date: 2026-06-16

## Summary

This release stabilizes live satellite audio with Doppler-corrected tuning on the Pluto Plus SDR and preserves the known-good NOAA Weather Radio diagnostic path.

## Functional changes

- Added Doppler-aware satellite Listen behavior.
- Added audio-owned Doppler tracking for live satellite audio.
- Added fallback from `radio_track.json` plan mode to active `radio_track_state.json` state mode.
- Added `doppler_state_fallback` response reporting for Listen start requests.
- Confirmed regular automatic Doppler tracking writes RX LO values through `iio_attr`.
- Confirmed Listen can start at the current active Doppler `rx_hz` when the regular tracker is already active.
- Preserved NOAA Weather Radio as fixed-frequency audio.
- Preserved the known-good `pluto_fm_receiver` helper binary during backend-only deploys.

## Verified behavior

The following checks were validated on the Pluto target:

- Backend starts after a real forced reboot.
- Running backend executable is not marked `(deleted)`.
- `/api/status` responds after reboot.
- Regular auto-Doppler tracking reports:
  - `state=auto_active`
  - `lo_path=iio_attr:ad9361-phy/altvoltage0/frequency`
  - `lo_write_result=written`
- Satellite Listen explicit Doppler start reports:
  - `doppler_track=true`
  - `doppler_state_fallback=true`
  - `audio_hz=<current Doppler rx_hz>`
- Audio track state reports:
  - `state=audio_active`
  - `plan_path=/mnt/jffs2/pluto_sat_tracker/data/radio_track_state.json`
  - `lo_write_result=written`
- NOAA diagnostic audio reports:
  - `noaa_fixed=true`
  - `doppler_track=false`
  - `doppler_state_fallback=false`
  - `audio_hz=162500000` for the 162.500 MHz test

## Deployment notes

Future Pluto deploy scripts should use this pattern:

1. Build backend locally in Docker.
2. Upload backend binary only unless the helper intentionally changed.
3. Run `sync`.
4. Force a real Pluto reboot.
5. Wait for SSH to go down and return.
6. Verify `/proc/<pid>/exe` is not marked `(deleted)`.
7. Verify `/api/status`.
8. Validate NOAA fixed-frequency audio after any audio change.
9. Validate satellite Doppler Listen against either `radio_track.json` or active `radio_track_state.json`.

## Important implementation notes

- The Pluto Plus firmware target uses `iio_attr -u local: -c ad9361-phy altvoltage0 frequency <hz>` for RX LO writes.
- `radio_track_state.json` fallback is required for real-pass Listen starts when the regular auto tracker is active but a full stored `radio_track.json` plan is not available.
- Manual kill/restart of `pluto_sat_tracker` is unreliable on this target because the process can continue running from a deleted binary inode. Use a real reboot.
