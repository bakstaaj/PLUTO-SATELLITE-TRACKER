# v2.8.16 Audio error capture

Adds a browser-side audio stream error capture panel for fast-moving `/api/radio/audio/live.wav` failures.

## Changes

- Wraps browser `fetch` to capture HTTP 409 responses from the live audio stream endpoint.
- Persists recent audio stream errors in `localStorage` under `pluto.audio.errors.v2816`.
- Exposes `window.plutoAudioErrors.list()`, `latest()`, `copyLatest()`, and `clear()` in the browser console.
- Shows a small copyable on-screen panel with the last audio stream error.

## Scope

- Web UI only.
- No backend C changes.
- No decoder behavior changes.
