# v2.8.8 - GMSK Phase 2 capture harness

This checkpoint adds a repeatable capture harness for GMSK/FSK demodulator development.

Marker: `GMSK_PHASE2_CAPTURE_HARNESS_V2_8_8`

## Scope

- Adds `tools/capture_gmsk_pcm_window_v2_8_8.py`.
- Captures the current Pluto live audio WAV stream during an active Receive/Listen session.
- Writes three offline artifacts per capture:
  - original WAV/raw stream
  - extracted signed 16-bit PCM
  - JSON metadata and text summary
- Computes diagnostics useful for the next demodulator phase:
  - RMS and peak
  - DC offset
  - clipping percentage
  - samples per symbol at assumed 9600 symbols/sec
  - zero-crossing estimate
  - transition activity proxy
  - coarse energy bins

## Non-goals

This does not claim GMSK packet decode, HDLC frame decode, AX.25 decode, or text recovery.
It also does not rebuild or replace the known-good deployed Pluto backend binary.

## Use during a pass

Start Receive/Listen in the browser during an active GMSK pass, then run:

```bash
python3 tools/capture_gmsk_pcm_window_v2_8_8.py --host 192.168.68.104 --seconds 8
```

Artifacts are written under:

```text
captures/gmsk_phase2/
```

These artifacts can be used to develop and tune symbol-clock recovery and slicer thresholds offline before adding backend frame/HDLC diagnostics.
