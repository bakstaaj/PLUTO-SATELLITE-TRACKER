Quick Start

Most Common Workflow

Use this order when you are starting fresh or after a Pluto reboot.

Setup

  1. Open the page and confirm Backend online.
  2. Set your station in Observer, or click the map to prefill latitude and longitude.
  3. Click Save Location.
  4. Click Sync Time.
  5. Click Refresh Catalog/TLE, then Regenerate Passes.

Operate

  1. Select a pass from Next Passes.
  2. Inspect the map, sky plot, and Doppler preview.
  3. Use Details to inspect pass timing, geometry, radio, and Doppler preview.
  4. Use Listen under the sky plot to start or stop analog audio for the selected pass.
Rotator Workflow

Safe Test Order

  1. Select the pass you want to follow from Next Passes.
  2. Confirm the live rotator target changes and Rotator source shows selected pass.
  3. Set rotator type, host, port, offsets, and elevation limits.
  4. Click Preview Command to verify the exact command without moving hardware.
  5. Use Test Move only after the preview looks correct.
  6. Use Start Rotator Tracking to follow the selected pass.
  7. Use Stop Tracking, Stop, or Park before changing hardware or settings.

Supported Protocols

  • Simulation is safest for workflow testing.
  • Hamlib rotctld TCP sends P <az> <el>.
  • EasyComm II sends AZ... EL....
  • Yaesu GS-232 sends Waaa eee.
  • SATRAN MK2/MK3 is pending until the firmware protocol is confirmed.
If multiple passes are active, selecting a pass publishes that pass plan and the rotator follows the selected pass.
Map And Focused Sample

Map Controls

  • Pan and zoom the map like a normal web map.
  • Click the map to prefill a new observer location.
  • Use Reset to clear the remembered map view.
  • Hover or click a pass sample to focus it.

Map Reading

  • The map shows the full visible pass path and the live observer look line.
  • The sky plot shows azimuth direction and elevation rings.
  • Select a pass from Next Passes to update the map and radio preview.
If Something Looks Wrong

Check These First

  1. Time shows synced.
  2. The selected pass is current and not stale.
  3. Radio Status shows a valid LO path.
  4. Refresh Status does not show a backend failure.

Best Status Panels

  • Time confirms Pluto UTC sync.
  • Radio Status shows tuned target, track point, and LO write result.
  • Selected Pass shows the buttons and Doppler preview for the current pass.
